Sok kérdés
-------------------
- Honnan jön a localhost:8080? Mire van a swaggerben a url szekció?
- Hogyan lehet a modelleket felannotálni (pl. JPA szerint)?
- Hogyan működik egy multi projekt?
- Java Runtime version váltás nem működik rendesen! settings -> pom.xml
- Mi ez a konstrukció: return findById(id).orElseThrow(() -> new Exception())
- A swagger-ben meghatározott constraint-ek (pl. format), a generált kódban hogyan jelentkezik?
- Az ident tab köz a két java forrás file-ban miért különbözik?
- Implementált vagy Override metódusok esetén is újra ki kell tenni az annotációkat?
- A generált kódban sok a nem használt osztály!
- Mitől lesz a response json formátum?

Projektek
--------------------
- daa-loginapi-demo
    Ismertető:
        Swagger-ből generált kód alapján készült RestApi implementáció. A generált kód és a megvalósítás egy projektben van.
        Nincsen benne perzisztencia, login állapotok nem kerülnek elmentésre. Nagyon primitív api.
    API szolgáltatások:
        /ping: válasz { "pong": "pong" }
        /login: "valaki"/"valami" login adatok esetén egy random session id-t ad vissza. Egyébként 404-es hiba, hibaüzenettel
        /logout/{session-id}: mindig 200-at ad vissza
    Tutorial:
        https://mydeveloperplanet.com/2022/02/08/generate-server-code-using-openapi-generator/
    Postman export file:
        MyFirstRequests.postman_collection.json

- spring-boot-openapi
    Ismertető:
        Minta demo tutorial Github projekt. Működik, ahogy a leírás állítja. Multi projekt: a generált kód és az Implementált
        üzleti logika külön projektben található (app, specification)
    Tutorial:
        https://reflectoring.io/spring-boot-openapi/

- daa-spring-boot-openapi
    Ismertető:
        A spring-boot-openapi szolgált mintának a saját swagger openapi generáláshoz. Lefordul, meg minden, de az összes
        api 404-es not found hibával tér vissza. Okokat nem sikerült kideríteni.
